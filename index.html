Description

          Designed by the engineers at Google, Go is a compiled, statically typed, language. Similar to C, it was designed to improve simplicity, and safety as opposed to other languages such as C. While Go has no objects or inheritance, you can use interfaces to declare a general set of methods for multiple files. This makes a semi form of inheritance, but not true inheritance. These all share common APIs. At the moment, Go's, most commonly used inside of Google. Outside of Google, the main usage falls under backend systems. Wrting HTTP and TCP severs in go provide a fast implementation which adds to its popularity. It consists of a Syntax environment that includes:

  • Optional variable declaration and initialization through type inference. I.E. x := 0
  • Fast compilation time: No use of interpreters.
  • Remote package management.

          Go's memory model lays out the conditions under which reads of a variable in one goroutine can be guranteed to observe values produced by writes to the same variable in a different goroutine. The goroutine is a lightweight thread managed by the Go runtime. Goroutines run in the same address space, this means the access to shared memory must be synchronized. Having a package sync inplementation provides basic synchronization primitives such as mutual exclusion locks.

          Go uses a package system which can be loosely compared to java's API system. Packages can be referenced by using the package name followed by the method: "io.Reader". The user can access remote repositories by using the "go get" command. Developers are envouraged to use a base path for package development. This reduces the likelihood of name collision when importing packages.

History

Contributors:

- Robert Griesemer (PHD in Computer Science)

- Rob Pike (Canadian born programmer and author)

- Ken Thompson (Designed and implemented the first Unix operating system)

- Ian Taylor (Author for Go frontend for GCC, the gold linker, and Taylor UUCP)

- Russ Cox (Optimized Go to run faster)

          Go was designed at Google in 2007, in response to criticisms with the languages that were in use by Google engineers at the time. The language design revolved around efficiency, Readability/Usability, and high-performance networking and multiprocessing. Robert Griesemer, Rob Pike, and Ken Thompson started gathering ideas for the language in September of 2007. By 2008, Ken started working on a compiler for the language which at the time generated C code as output. By mid 2007, the language had become a full-time project and had developed to the point where construction of a production compiler was in their sights. In May 2008 Ian Taylor started on a GCC front end for Go using the team’s ideas and draft specifications. Russ Cox joined the team in 2008 and helped translate the language draft and its libraries into a functional language. At the time of Go’s creation, the two main languages to write servers were C++ and java. Go was designed to reduce the repetition of these languages, the idea was to have efficiency, fluidity, and safety in a single language. The list below contains examples of well-known companies whose applications were either completely, or partially programmed in Go:
