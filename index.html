Description

         

The Go programming language, which is often referred to as GoLang) is a compiled language developed by Google. They made the Go language with the intention of giving programmers a way to create efficient and dependable software. The other programming language that Go is most like would be C, since they are both statically typed and very similar syntactically. Unlike C though, Go was developed with features such as garbage collection, memory safety, and structural typing.

This language is quick in compiling and has a fast execution speed. Contributing to this is Go's goroutines which work as threads and enable concurrent processing. Because of this fast speed, it's like an interpreted language.

Go was also built to be easy to read and work with. It has tools like Gofmt that will take your code and automatically indent and format it for you. This provides a simple way to keep your code in order and keep the readability seamless. Another built in tool that makess this language great is 'Go run' which will automatically and simultaneously compile and run your code (super cool!). Go also has an extensive list of built in libraries you can use to keep programs simple and easy.

With all the features of this language, it is great to use for:

- Networking (webservice, cloud based apps)

- Systems programming (especially since its similar to C)

- Microservices (an app devided up into many services/unique processes)

- Programs that need concurrency or threading

- Distributed Systems (independent computers linked by a single network)

History

Contributors:

- Robert Griesemer (PHD in Computer Science)

- Rob Pike (Canadian born programmer and author)

- Ken Thompson (Designed and implemented the first Unix operating system)

- Ian Taylor (Author for Go frontend for GCC, the gold linker, and Taylor UUCP)

- Russ Cox (Optimized Go to run faster)

          Go was designed at Google in 2007, in response to criticisms with the languages that were in use by Google engineers at the time. The language design revolved around efficiency, Readability/Usability, and high-performance networking and multiprocessing. Robert Griesemer, Rob Pike, and Ken Thompson started gathering ideas for the language in September of 2007. By 2008, Ken started working on a compiler for the language which at the time generated C code as output. By mid 2007, the language had become a full-time project and had developed to the point where construction of a production compiler was in their sights. In May 2008 Ian Taylor started on a GCC front end for Go using the team’s ideas and draft specifications. Russ Cox joined the team in 2008 and helped translate the language draft and its libraries into a functional language. At the time of Go’s creation, the two main languages to write servers were C++ and java. Go was designed to reduce the repetition of these languages, the idea was to have efficiency, fluidity, and safety in a single language. Specification

Core Features:

Standard Library - Powerful library of packages to support Go developers.

Supports Type Safety - Compiler will catch errors before execution.

Garbage Collector - Frees the programmer from manually dealing with memory deallocation.

Syntax - Similar to C.

GoDoc - Analysis tool that extracts and generates documentation for Go programns.

GoRoutines - Functions or methods that run concurrently. Which means that a program does not need to wait for a function to complete in order to move on to the next task.
